# Palo IT Document Styling Guide

## 📚 Overview

This guide reflects Palo IT's modern, minimal brand identity as seen on palo-it.com/en/. 
The focus is on clean, professional design with emphasis on content over decoration. 
Palo IT positions itself as a tech-forward, enterprise AI consulting firm with a sleek, contemporary aesthetic.

## 🎨 Brand Color Palette

### Primary Colors
| Color | Use | RGB | Hex |
|-------|-----|-----|-----|
| **Palo IT Black** | Primary text, headers, main content, logo | 0, 0, 0 | #000000 |
| **Palo IT White** | Primary background, text on dark | 255, 255, 255 | #FFFFFF |
| **Light Gray** | Subtle backgrounds, borders, dividers | 240, 240, 240 | #F0F0F0 |
| **Medium Gray** | Secondary text, captions, disabled states | 128, 128, 128 | #808080 |
| **Dark Gray** | Alternative text color, subtle emphasis | 64, 64, 64 | #404040 |

### Accent Colors (Used Sparingly)
- **Tech Blue**: #0066CC (hyperlinks, interactive elements, call-to-action highlights)
- **Gen-e2 Purple**: #7B3FF2 (Gen-e2™ trademark references, premium features)
- **Success Green**: #00A651 (positive indicators, success states)
- **Warning Orange**: #FF8C42 (alerts, important notices - use minimally)

## 🔤 Typography System

### Font Families
**Primary Family (Headlines & Body)**: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif
(System fonts optimized for clarity and tech industry standards - see www.palo-it.com for reference)

**Secondary Family (Premium)**: "Inter", "Helvetica", "Arial" (clean, modern sans-serif)

**Monospace (Data/Code)**: "Monaco", "Fira Code", "Courier New" (for technical content, commands, code snippets)

### Font Sizes & Usage

| Size | Usage | Example | Weight |
|------|-------|---------|--------|
| **40-48pt** | Main title / Document headline | "Gen-e2™: Enterprise Grade ADLC" | 600-700 (semibold/bold) |
| **28-32pt** | Section headers / Major headings | "What Makes Us Different" | 600 (semibold) |
| **18-20pt** | Sub-headers / Feature titles | "AI Expertise", "Proven Enterprise ROI" | 600 (semibold) |
| **14-16pt** | Body text / Primary content | Standard paragraph text | 400 (regular) |
| **12-14pt** | Secondary content, descriptions | Feature descriptions, captions | 400 (regular) |
| **11-12pt** | Fine print, metadata | Dates, sources, disclaimers | 400 (regular) |
| **10pt** | Annotations, footnotes | "[source: palo-it.com]" | 400 (regular) |

### Text Styling Rules

**Bold (600-700 weight)** - Used for:
- Section titles
- Key metrics, statistics, numbers
- Important feature names
- Call-to-action text
- Definition terms

**Regular (400 weight)** - Primary weight for all content
- Ensures clean, minimal appearance
- Better readability at all sizes
- No excessive emphasis

**Italic** - Used sparingly for:
- Product names (Gen-e2™, first mention only)
- Emphasis on key business outcomes
- Citations/sources

**ALL CAPS** - Avoid entirely
- Contradicts modern minimal aesthetic
- Use title case instead for headers
- Exception: Product trademark (Gen-e2™)

## 📐 Layout & Spacing

### Digital-First Formats
Palo IT's modern approach emphasizes responsive, digital-first layouts:
- **Web/Screen** (Primary): 1440px-1920px wide
- **PDF Documents**: 8.5" × 11" (Letter) or A4
- **Mobile** (Secondary): 375px-540px wide
- **Presentation**: 16:9 widescreen (if needed, but web-first recommended)

### Margin & Padding Guidelines

**For Digital Content**:
```
┌────────────────────────────┐
│ 40-60px padding            │
│ ┌──────────────────────┐   │
│ │   CONTENT AREA       │   │  Max width: 1200px
│ │                      │   │  
│ │                      │   │
│ └──────────────────────┘   │
│ 40-60px padding            │
└────────────────────────────┘
```

**For PDF Documents**:
```
┌─────────────────────────────┐
│  0.5" - 0.75"               │
│  ┌───────────────────────┐  │  
│  │   CONTENT AREA        │  │  
│  │                       │  │  
│  └───────────────────────┘  │
│  0.5" - 0.75"               │
└─────────────────────────────┘
```

### Content Spacing Standards
- **Header to Body**: 20-30px (digital) / 0.25" (print)
- **Between Paragraphs**: 15-20px (digital) / 0.15" (print)
- **Between Sections**: 30-40px (digital) / 0.3" (print)
- **List Item Spacing**: 12-15px vertical
- **List Indent**: 20px per level (digital), 0.25" (print)
- **Line Height**: 1.5-1.6 for body text (26-28px for 16-18pt font)

## 📊 Content Patterns

### Pattern 1: Hero / Title Section
Modern, minimal title presentation for documents, webpages, or presentations.

```
┌─────────────────────────────────┐
│                                 │
│         Document Title          │  36-48pt bold black text
│                                 │
│  Subtitle or Date               │  18-20pt regular gray text
│                                 │
│  Optional: Brief context line   │  14-16pt regular gray
│                                 │
└─────────────────────────────────┘
```

**Design Notes**:
- White or light gray background
- Centered text alignment
- No colored boxes or borders
- Optional: subtle horizontal line below title
- Company logo (top-left or bottom-right, faint)

**Design Notes**:

- "Learn More" link in #0066CC (tech blue)
- Rounded corners (4-8px)
- Hover state: slight shadow increase, text color to #0066CC

## 🖼️ Visual Elements

### Images & Graphics
- **Frequency**: 1 image per 2-3 content sections (not every slide)
- **Placement**: Right-aligned (web), or full-width for emphasis
- **Size**: 40-60% of container width (responsive)
- **Borders**: None, or very subtle 1px #E0E0E0 for definition
- **Style**: Professional photography or clean data visualizations
- **Captions**: 12-14pt gray text below (optional)
- **Aspect Ratio**: 16:9 or 4:3 preferred (avoid extreme proportions)

### Icons & Visual Markers
- **Library**: Heroicons, Feather Icons, or similar modern icon sets
- **Size**: 24×24px (body), 40-60px (feature cards), 80-120px (hero sections)
- **Color**: Inherit from text color (black/gray), or #0066CC for interactive elements
- **Style**: Outline/minimal style (not filled or decorative)
- **Usage**: Sparingly, for visual rhythm only - not essential for comprehension

### Dividers & Borders
- **Type**: Thin 1px horizontal lines in #E0E0E0
- **Usage**: Between sections (not within sections)
- **Spacing**: 30-40px above and below divider
- **Full-width**: Occasionally for major section breaks
- **Alternative**: Increased whitespace instead of visible borders

### Tables & Data Visualization
- **Max Columns**: 4-5 (wider = less readable)
- **Max Rows**: 5-7 body rows + header (use pagination or scrolling for larger datasets)
- **Header**: Dark gray or black background (#333 or #000), white text
- **Borders**: 1px #E0E0E0 between cells
- **Row Colors**: Alternating white and #F0F0F0 for readability
- **Data Labels**: Show values directly on charts when possible (bars, lines, slices)
- **Chart Types**: Bar, line, pie, scatter (simple, no 3D effects)
- **Font**: Body text size or smaller (12-14pt for labels)

### Colors in Visualizations
- **Primary Data**: #0066CC (tech blue)
- **Secondary Data**: #808080 (medium gray)
- **Success/Positive**: #00A651 (green)
- **Warning/Alert**: #FF8C42 (orange)
- **Neutral**: #E0E0E0 (light gray)
- **Avoid**: Bright neon colors, excessive color variation

## ✅ Quality Checklist

### Before Finalizing Any Document:

- [ ] **Colors**: Only uses grayscale + tech blue (#0066CC) + accent colors sparingly
- [ ] **Fonts**: System fonts or clean sans-serif (no serif fonts for body)
- [ ] **Font Sizes**: Consistent hierarchy (40-48pt > 28-32pt > 18-20pt > 14-16pt > 12-14pt)
- [ ] **Text Weight**: Regular (400) for body, semibold (600) for headers only
- [ ] **Spacing**: Even margins (40-60px), consistent padding between elements
- [ ] **Emphasis**: Bold weight for headers only, italic used sparingly
- [ ] **Images**: One every 2-3 content sections, properly sized and optimized
- [ ] **Content Density**: Max 20-25 lines of text per page/section
- [ ] **Alignment**: Left-aligned body text (centered only for titles/CTAs)
- [ ] **Visual Balance**: Alternating text-heavy and visual-rich sections
- [ ] **Consistency**: Same pattern type looks identical across document
- [ ] **Responsiveness**: Layouts work on mobile (375px) and desktop (1440px+)
- [ ] **Accessibility**: Color contrast ≥4.5:1, no color-only indicators
- [ ] **Whitespace**: Generous margins and breathing room (not cramped)
- [ ] **Call-to-Action**: Clear, minimal CTAs with consistent styling
- [ ] **Branding**: Logo placement consistent, Gen-e2™ trademark used correctly

## 🔧 Implementation Guide

### For Digital-First Documents (Preferred)

**1. Web/HTML Documents**:
- Use system fonts (-apple-system, BlinkMacSystemFont, "Segoe UI")
- Max content width: 1200px (centered container)
- Responsive breakpoints: 375px (mobile), 768px (tablet), 1024px (desktop)
- Line height: 1.5-1.6 for body text
- Horizontal rhythm: 8px baseline grid

**2. Color Implementation**:
```css
/* CSS Variables (Recommended) */
:root {
  --color-black: #000000;
  --color-white: #FFFFFF;
  --color-gray-light: #F0F0F0;
  --color-gray-medium: #808080;
  --color-gray-dark: #404040;
  --color-blue-tech: #0066CC;
  --color-blue-gen-e2: #7B3FF2;
  --color-green-success: #00A651;
  --color-orange-warn: #FF8C42;
}
```

**3. Typography Stack**:
```css
body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", 
               Roboto, "Helvetica Neue", Arial, sans-serif;
  font-size: 16px;
  line-height: 1.6;
  color: #333333;
}

h1 { font-size: 48px; font-weight: 600; }
h2 { font-size: 32px; font-weight: 600; }
h3 { font-size: 20px; font-weight: 600; }
```

**4. Component Spacing**:
- Margins: 40px (mobile), 60px (desktop)
- Section padding: 40px top/bottom (mobile), 60px (desktop)
- Element gap: 20px (mobile), 30px (desktop)

---

### For PDF Documents

**1. Setup**:
- Page size: A4 or Letter (8.5" × 11")
- Margins: 0.5"-0.75" on all sides
- Font embedding: Use system fonts or web-safe alternatives

**2. Layout**:
- Single or two-column layout (not more)
- Max characters per line: 70-80
- Line height: 1.5 for body, 1.2 for headers

**3. Page Structure**:
```
├── Title Page (optional)
├── Table of Contents (if >10 pages)
├── Content Sections
└── Back Matter (optional: appendix, glossary)
```

**4. Image Handling**:
- Resolution: 72-150 DPI (web), 300 DPI (print)
- Compression: Optimize before embedding
- Sizing: Full-width or 50-60% width columns

---

### For PowerPoint / Presentation Slides (If Required)

> **Note**: Palo IT's primary format is web/digital. Use PowerPoint only when necessary for client delivery.

**1. Slide Setup**:
- Aspect ratio: 16:9 widescreen
- Slide size: 13.3" × 7.5"
- Master slides: Use minimal template (white background, black text)

**2. Color Scheme**:
- Background: White (#FFFFFF)
- Text: Black (#000000) or Dark Gray (#333333)
- Accents: Tech Blue (#0066CC) only for interactive elements
- Avoid full-screen color blocks (use subtle header bar if needed)

**3. Slide Types**:
- Title slide: Large title (40-48pt) + subtitle
- Content: Header + body text + optional image
- Feature: 3-4 cards in grid layout
- Data: Chart/table with minimal decoration
- Closing: CTA or thank you slide

**4. Animation** (Use Minimally):
- Fade in/out only (no bounces or spins)
- Timing: 300-500ms
- Avoid: Distracting animations that detract from content

### Accessibility Standards

**Color Contrast**:
- Minimum WCAG AA: 4.5:1 (normal text), 3:1 (large text)
- Black on white: 21:1 (excellent)
- Tech blue on white: 8.6:1 (excellent)

**Text & Navigation**:
- Readable fonts: System fonts, sans-serif
- Font sizes: Min 14px body, 18px headers
- Link styling: Underline or distinct color (#0066CC)
- Focus indicators: Visible keyboard navigation

**Images & Media**:
- Alt text: Descriptive for all images
- Captions: Provide for videos/complex graphics
- Charts: Provide data tables as fallback

**Keyboard Navigation**:
- Tab order: Logical and predictable
- Skip links: "Skip to main content" link visible on first Tab
- Interactive elements: All keyboard-accessible
